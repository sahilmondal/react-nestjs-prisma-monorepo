# AI Agent Developer Guide

This repository contains a full-stack monorepo featuring a React (Vite + TanStack Router) frontend and a NestJS + Prisma backend. This guide serves as the authoritative context for AI agents working in this codebase.

## Architectural Overview

### 1. `apps/http-api`

The backend API built with NestJS.

- **ORM**: Prisma (schema in `apps/http-api/prisma/schema.prisma`).
- **Auth**: Passport.js for JWT & OAuth.
- **Rules**: Never bypass the Auth Guards. Ensure validation decorators are applied to all incoming DTOs.

### 2. `ui/apps/react-app`

The primary frontend application.

- **Framework**: React 19 + Vite + TanStack Router (`@tanstack/react-router`).
- **Styling**: Tailwind CSS v4. No Next.js or generic CSS files; styling happens via Tailwind utility classes and CSS variables in `styles.css`.
- **Routing**: Uses file-based routing in `src/routes`.
- **Local Components**: Application-specific components like the `Auth` feature (login, signup, forgot-password forms, and the Auth Guard) are located in `src/components/Auth`.

### 3. `packages/auth-client`

A shared client library for making API requests to the NestJS backend and managing session state.

- **State**: Uses `zustand` to manage the JWT token and user session (`useAuthStore`).
- **API Wrapper**: Defines async functions to call backend endpoints.
- **Env**: Extracts public environment variables (e.g., `VITE_API_URL`).

### 4. `ui/shared/*`

Reusable UI components.

- **`ui-core`**: The base shadcn/ui components (`Button`, `Input`, `Form`, etc.).
- **`ui-components`**: Higher-level domain components that are shared across apps (if any).

## Working with the Frontend

1. **Routing**: When creating a new page, add a file in `ui/apps/react-app/src/routes`. `routeTree.gen.ts` is automatically generated by TanStack Router.
2. **Navigation**: Import `Link` or use `useNavigate` from `@tanstack/react-router`. **DO NOT** use `next/link` or `next/navigation`.
3. **State Management**: If a piece of state needs to be global, update the Zustand store in the relevant package. For local component state, use React `useState`.
4. **Environment Variables**: Use the `VITE_` prefix for any variable that needs to be exposed to the client.

## Scripts & CLI Commands

- `bun install`: Installs dependencies and links all workspace packages across the monorepo.
- `bun run typecheck`: Runs `tsc --noEmit` on all workspace packages. Use this often to verify refactors.
- `bun run dev`: Concurrently starts the NestJS API and the Vite React server.

## Code Standards

- **Absolute Imports**: Rely on configured TypeScript paths (e.g., `@workspace/ui-core`, `@workspace/auth-client`) rather than relative imports that span outside a package's directory.
- **Zod Validation**: We prefer Zod for all client-side schema validations (e.g., in React Hook Form).
