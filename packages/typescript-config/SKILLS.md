# TypeScript Config - Shared Compiler Settings

Shared TypeScript configurations for consistent compiler behavior across all apps and packages. Provides base, Next.js, and React library specific configurations.

**Location**: `packages/typescript-config/`  
**Package Manager**: Bun

## Configuration Files

### 1. Base Configuration (`base.json`)

Baseline TypeScript settings for all projects:

```json
{
  "compilerOptions": {
    "target": "ES2022",
    "useDefineForClassFields": true,
    "lib": ["ES2022", "DOM", "DOM.Iterable"],
    "module": "ESNext",
    "skipLibCheck": true,
    "strict": true,
    "noUnusedLocals": true,
    "noUnusedParameters": true,
    "noImplicitReturns": true,
    "isolatedModules": true,
    "esModuleInterop": true,
    "allowSyntheticDefaultImports": true,
    "resolveJsonModule": true,
    "declaration": true,
    "declarationMap": true,
    "sourceMap": true,
    "baseUrl": ".",
    "moduleResolution": "bundler"
  },
  "include": ["src"],
  "exclude": ["node_modules", "dist", ".next"]
}
```

**Key Options**:

| Option               | Value   | Purpose                                           |
| -------------------- | ------- | ------------------------------------------------- |
| `target`             | ES2022  | Modern JavaScript target (Node 16+, all browsers) |
| `module`             | ESNext  | Preserves import/export syntax                    |
| `strict`             | true    | All strict type checks enabled                    |
| `noUnusedLocals`     | true    | Error on unused variables                         |
| `noUnusedParameters` | true    | Error on unused function params                   |
| `noImplicitReturns`  | true    | Error if not all code paths return                |
| `isolatedModules`    | true    | Each file can be transpiled independently         |
| `moduleResolution`   | bundler | Modern Node.js-compatible resolution              |
| `declaration`        | true    | Generate `.d.ts` files (library output)           |
| `sourceMap`          | true    | Enable debugger support                           |

### 2. Next.js Configuration (`nextjs.json`)

Extended config for Next.js applications:

```json
{
  "extends": "./base",
  "compilerOptions": {
    "target": "ES2022",
    "lib": ["ES2022", "DOM", "DOM.Iterable"],
    "jsx": "preserve",
    "plugins": [
      {
        "name": "next"
      }
    ]
  },
  "include": ["next-env.d.ts", "**/*.ts", "**/*.tsx"],
  "exclude": ["node_modules"]
}
```

**Key Additions**:

- `jsx: "preserve"` — Next.js handles JSX transpilation
- `next` plugin — Provides Next.js-specific types
- Includes `next-env.d.ts` (auto-generated by Next.js)

**Usage** (for `apps/nextjs-app`):

```json
{
  "extends": "@workspace/typescript-config/nextjs"
}
```

### 3. React Library Configuration (`react-library.json`)

Extended config for React component libraries:

```json
{
  "extends": "./base",
  "compilerOptions": {
    "jsx": "react-jsx",
    "target": "ES2022",
    "lib": ["ES2022", "DOM", "DOM.Iterable"],
    "declaration": true,
    "declarationDir": "dist",
    "outDir": "dist",
    "rootDir": "src"
  },
  "include": ["src"],
  "exclude": ["node_modules", "dist", "**/*.test.ts", "**/*.test.tsx"]
}
```

**Key Additions**:

- `jsx: "react-jsx"` — React 17+ JSX transform (no import needed)
- `declaration: true` — Generate `.d.ts` for library consumers
- `outDir: "dist"` — Compiled output location
- Excludes test files

**Usage** (for `packages/ui`):

```json
{
  "extends": "@workspace/typescript-config/react-library"
}
```

## Strict Mode Options Explained

### `noUnusedLocals`

```typescript
// ❌ Error
const userId = 123;
console.log("Loaded");

// ✅ OK
const userId = 123;
console.log(userId);

// ✅ OK (underscore prefix)
const _userId = 123;
console.log("Loaded");
```

### `noUnusedParameters`

```typescript
// ❌ Error
function getUserId(userId, userName) {
  return userId; // userName unused
}

// ✅ OK
function getUserId(userId, _userName) {
  return userId;
}

// ✅ OK
function getUserId(userId) {
  return userId;
}
```

### `noImplicitReturns`

```typescript
// ❌ Error
function validate(email: string): boolean {
  if (email.includes('@')) {
    return true;
  }
  // Missing return for non-@ emails
}

// ✅ OK
function validate(email: string): boolean {
  if (email.includes('@')) {
    return true;
  }
  return false;
}
```

### `strict` (Umbrella)

Enables all strict checks:

- `strictNullChecks` — Can't assign null/undefined to non-nullable types
- `strictFunctionTypes` — Strict function type checking
- `strictBindCallApply` — Strict .bind(), .call(), .apply()
- `strictPropertyInitialization` — Class properties must be initialized
- `noImplicitThis` — `this` must have explicit type
- `alwaysStrict` — Parse in strict mode, emit `"use strict"`

**Example (strictNullChecks)**:

```typescript
// ❌ Error
const value: string = null;

// ✅ OK
const value: string | null = null;

// ✅ OK
const value: string = "default";
```

{
  "extends": "@workspace/typescript-config/base",
  "compilerOptions": {

    "rootDir": "./src",
    "experimentalDecorators": true,
    "emitDecoratorMetadata": true,

**Additions**:

- `emitDecoratorMetadata` — Required for NestJS DI to work

## Common Type Patterns

### Function Parameter Types

```typescript
// ❌ Implicit any
function login(email, password) {
  return email;
}

// ✅ Explicit types
function login(email: string, password: string): Promise<User> {
  return authenticate(email, password);
}
```

### Generic Types

```typescript
// API response wrapper
interface ApiResponse<T> {
  data: T;
  status: number;
}

// Usage
const response: ApiResponse<User> = {
  data: { id: '1', email: 'user@example.com' },
  status: 200,
};
```

### Union Types

```typescript
// Multiple possible types
type AuthStatus = 'IDLE' | 'LOADING' | 'AUTHENTICATED' | 'UNAUTHENTICATED';

const status: AuthStatus = 'LOADING'; // ✅ OK
const status: AuthStatus = 'INVALID'; // ❌ Error
```

### Optional Properties

```typescript
interface User {
  id: string;
  email: string;
  name?: string; // Optional
}

const user: User = {
  id: '1',
  email: 'user@example.com',
  // name is optional
};
```

### Discriminated Unions

```typescript
type Result =
  | { status: 'success'; data: User }
  | { status: 'error'; message: string };

function handle(result: Result) {
  if (result.status === 'success') {
    console.log(result.data.email); // ✅ data available here
  } else {
    console.log(result.message); // ✅ message available here
  }
}
```

## Migration to Strict Mode

If enabling strict mode on existing code:

1. **Start with least strict**:

   ```json
   {
     "strict": false,
     "noImplicitAny": true
   }
   ```

2. **Fix `noImplicitAny` errors** (add type annotations)

3. **Enable `strictNullChecks`**:

   ```json
   {
     "noImplicitAny": true,
     "strictNullChecks": true
   }
   ```

4. **Fix null/undefined errors** (check before use)

5. **Enable full strict mode**:
   ```json
   {
     "strict": true
   }
   ```

## Troubleshooting

### Issue: "Cannot find module '@workspace/...'"

**Solution**: Check `tsconfig.json` path resolution:

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@workspace/*": ["*"]
    }
  }
}
```

### Issue: `declaration` files not generated

**Solution**: Ensure `declaration: true`:

```json
{
  "compilerOptions": {
    "declaration": true,
    "declarationDir": "dist"
  }
}
```

### Issue: JSX errors in Next.js

**Solution**: Use `jsx: "preserve"` (Next.js handles transpilation):

```json
{
  "compilerOptions": {
    "jsx": "preserve"
  }
}
```
